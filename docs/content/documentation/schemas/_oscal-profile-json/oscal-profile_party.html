Party (organization or person): party object

This object appears unlabelled in any array called parties

A responsible entity, either singular (an organization or person) or collective (multiple persons)

The party object has the following members (properties):

  • id property (optional)
  • role-id property (optional)
  • An array labelled persons, containing person objects (optional)
  • org object (optional)
  • notes object (optional)

As contents one of org or person is required, or both; there may be only one org, but many persons.

Note that persons can also be indicated with organizational affiliations by using orgname inside person. Contact information in that case belongs to the person, not the organization: use org when an organization as such serves as a documented party.

This object appears as a property on: metadata.
