        OSCAL System Security Plan (SSP) Model XML Schema Reference

        Schema download oscal_ssp_schema.xsd

        Schema version 1.0.0-milestone2

        Short name oscal-ssp

        XML namespace http://csrc.nist.gov/ns/oscal/1.0

        The OSCAL Control SSP format can be used to describe the information typically specified in a system security plan, such as those defined in NIST SP 800-18.

        The root of the OSCAL System Security Plan (SSP) format is system-security-plan.

        addr-line

        Name Address line

        Description A single line of an address.

        An element (string)

        This element appears inside: address.

        address

        Name Address

        Description A postal address.

        This element appears inside: person and org.

        The address element has the following contents (in order):

        • type attribute (string) [optional] Indicates the type of address.

          The value may be locally defined, or one of the following:

          • home: A home address.
          • work: A work address.

        • addr-line elements (string) [0 to ∞] A single line of an address.

        • city element (string) [0 or 1] City, town or geographical region for mailing address

        • state element (string) [0 or 1] State, province or analogous geographical region for mailing address

        • postal-code element (string) [0 or 1] Postal or ZIP code for mailing address

        • country element (string) [0 or 1] Country for mailing address

        adjustment-justification

        Name Adjustment Justification

        Description If the selected security level is different from the base security level, this contains the justification for the change.

        An element (markup-multiline)

        This element appears inside: confidentiality-impact, integrity-impact, and availability-impact.

        algorithm

        Name Hash algorithm

        Description Method by which a hash is derived

        An attribute (string)

        The value may be locally defined, or one of the following:

        • SHA-224
        • SHA-256
        • SHA-384
        • SHA-512
        • RIPEMD-160

        This attribute appears on: hash.

        Remarks

        Valid values MUST be a value defined in the W3C XML Security Algorithm Cross-Reference Digest Methods (W3C, April 2013).

        annotation

        Name Annotation

        Description A name/value pair with optional explanatory remarks.

        This element appears inside: party, person, org, role, responsible-party, system-characteristics, system-information, leveraged-authorization, authorization-boundary, network-architecture, data-flow, system-implementation, user, component, service, interconnection, inventory-item, implemented-component, implemented-requirement, responsible-role, and by-component.

        The annotation element has the following contents:

        • name attribute (NCName) [required]

        • id attribute (string) [optional]

        • ns attribute (string) [optional]

        • value attribute (string) [optional] Indicates the value of the characteristic.

        • remarks element (markup-multiline) [0 or 1] Additional commentary on the parent item.

        Remarks

        Annotations permit the deployment and management of arbitrary controlled values for any purpose useful to an application or implementation scenario.

        The value (flag) of an annotation is a simple scalar value or list of values. The body of the annotation offers remarks in explanation of or reference to this value.

        availability-impact

        Name Availability Impact Level

        Description The expected level of impact resulting from the disruption of access to or use of information or the information system.

        This element appears inside: information-type.

        The availability-impact element has the following contents (in order):

        • prop elements (string) [0 to ∞] A value with a name, attributed to the containing control, part, or group.

        • base element (string) [1] The prescribed base (Confidentiality, Integrity, or Availability) security impact level.

          The value must be one of the following:

          • fips-199-low: A 'low' sensitivity level as defined in FIPS-199.
          • fips-199-moderate: A 'moderate' sensitivity level as defined in FIPS-199.
          • fips-199-high: A 'high' sensitivity level as defined in FIPS-199.

        • selected element (string) [0 or 1] The selected (Confidentiality, Integrity, or Availability) security impact level.

          The value must be one of the following:

          • fips-199-low: A 'low' sensitivity level as defined in FIPS-199.
          • fips-199-moderate: A 'moderate' sensitivity level as defined in FIPS-199.
          • fips-199-high: A 'high' sensitivity level as defined in FIPS-199.

        • adjustment-justification element (markup-multiline) [0 or 1] If the selected security level is different from the base security level, this contains the justification for the change.

        back-matter

        Name Back matter

        Description A collection of citations and resource references.

        This element appears inside: system-security-plan.

        The back-matter element has the following contents (in order):

        • citation elements [0 to ∞] A citation to resources, either external or internal (by means of internal cross-reference).

        • resource elements [0 to ∞] A resource associated with the present document.

        Remarks

        Provides a collection of identified citation and resource objects that can be referenced by a link with a rel value of "reference" and an href value that is a fragment "#" followed by a reference to a citation identifier. A citation can reference a resource by specifying a local target value using the same fragment-identifier approach.

        The following is a contrived example to show the use of link, citation, and resource.

        <profile>
<metadata>
<link rel="reference" href="#citation1">My citation</link></metadata>
<back-matter>
<citation id="citation1">
<target>#resource1</target></citation>
<resource id="resource1">
<rlink href="https://example.org/some-resource"></rlink></resource></back-matter></profile>

        citation

        Name Citation

        Description A citation to resources, either external or internal (by means of internal cross-reference).

        This element appears inside: back-matter.

        The citation element has the following contents (in order):

        • id attribute (string) [required]

        • target elements (string) [0 to ∞] An address for retrieval of a citation

        • title element (markup-line) [0 or 1] A title for display and navigation

        • desc element (string) [0 or 1] A short textual description

        • doc-id elements (string) [0 to ∞] A document identifier qualified by an identifier type.

        • prop elements (string) [0 to ∞] A value with a name, attributed to the containing control, part, or group.

        • Any element (in a foreign namespace)

        Remarks

        The structure of a citation is not intended to be a full biblographic representation of a cited resource. Instead, it is expected that matching to a source of bibliograpical information will be handled by use of a doc-id. Use of a canonical identifier such as a DOI is recommended for any citation.

        Within this element, a generic title can cite a title. If citation information takes the form of a bibliographic entry, then desc can be used to provide it.

        component

        Name Component

        Description A defined component that can be part of an implemented system.

        This element appears inside: system-implementation.

        The component element has the following contents (in order):

        • id attribute (NCName) [required] A unique identifier for a component.

        • name attribute (string) [required] The component's human-readable name.

        • component-type attribute (string) [required]

          The value may be locally defined, or one of the following:

          • software: Any software, operating system, or firmware
          • hardware
          • service: A service that may provide APIs.
          • policy
          • process
          • procedure
          • plan
          • guidance: Any guideline or recommendation
          • standard
          • validation

        • description element (markup-multiline) [1] A description of the component, including information about its function.

        • prop elements (string) [0 to ∞] A value with a name, attributed to the containing control, part, or group.

          Some specific property names can be assigned to the component at large, or can be defined when a component is associated with an inventory-item within the implemented-component construct.

        • annotation elements [0 to ∞] A name/value pair with optional explanatory remarks.

          Some specific annotation names can be assigned to the component at large, or can be defined when a component is associated with an inventory-item within the implemented-component construct.

        • link elements (markup-line) [0 to ∞] A reference to a local or remote resource

        • status element [1] Describes the operational status of the system.

        • responsible-role elements [0 to ∞] Defines a role that has responsibility for the component.

          A role defined in this way can be associated with a party or parties here, or where the component is referenced (e.g., inventory-item, implemented-requirement/by-component).

          The roles "maintainer" or "provider" can be used to define the supplier. In such cases party-id(s) can be used to associate the providing organization. For products, services, and APIs, this is the vendor and/or retailer information. For policies, processes, plans, standards, and guidance, this is the authoring organization.

        • remarks element (markup-multiline) [0 or 1] Additional commentary on the parent item.

        Remarks

        Components may be products, services, application programming interface (APIs), policies, processes, plans, guidance, standards, or other tangible items that enable security and/or privacy.

        The type indicates which of these component types is represented.

        diagram

        Name Diagram

        Description A graphic that provides a visual representation the system, or some aspect of it.

        This element appears inside: authorization-boundary, network-architecture, and data-flow.

        The diagram element has the following contents (in order):

        • id attribute (NCName) [required] An identifier for this diagram.

        • description element (markup-multiline) [0 or 1] A description of the diagram (e.g., alternate text). This can be used to support compliance with requirements from Section 508 of the United States Workforce Rehabilitation Act of 1973.

        • prop elements (string) [0 to ∞] A value with a name, attributed to the containing control, part, or group.

        • link elements (markup-line) [0 to ∞] A reference to a local or remote resource

        • caption element (markup-line) [0 or 1] A brief caption to annotate the diagram.

        • remarks element (markup-multiline) [0 or 1] Commentary about the diagram that enhances it.

        Remarks

        A diagram must include a link with a rel value of "diagram", who's href references a remote URI or an internal reference within this document containing the diagram.

        The internal reference "#diagram1" points to an attached resource defined in the back-matter as a resource. The media-type indicates that the image is a Portable Network Graphics (PNG) image.

        <diagram id="boundary-diagram-1">
<description>A boundary diagram.</description>
<link rel="diagram" href="#diagram1" media-type="image/png"></link></diagram>

        doc-id

        Name Document Identifier

        Description A document identifier qualified by an identifier type.

        An element (string)

        This element appears inside: metadata and citation.

        The doc-id element must have the attribute:

        • type attribute (string) [required] Qualifies the kind of document identifier.

          The value may be locally defined, or one of the following:

          • doi: A Digital Object Identifier (DOI); use is preferred, since this allows for retrieval of a full bibliographic record.

        Remarks

        A document identifier or name, qualified by a type that indicates the naming system of the identifier. A document identifier may provide a hint to the persistent location of the same or another version of the document.

        Specific usage rules, nomenclatures, and recognized file types (e.g., DOIs, CMS locators, etc.) may be defined at application level. The value of this field may also be a hash or checksum.

        hash

        Name Hash

        Description A representation of a cryptographic digest generated over a resource using a hash algorithm.

        An element (string)

        This element appears inside: rlink.

        The hash element must have the attribute:

        • algorithm attribute (string) [required]

          The value may be locally defined, or one of the following:

          • SHA-224
          • SHA-256
          • SHA-384
          • SHA-512
          • RIPEMD-160

        Remarks

        A hash value can be used to authenticate that a referenced resource is the same resources as was pointed to by the author of the reference.

        When appearing as part of a resource (data object), the hash should apply to the referenced object.

        implemented-component

        Name Implemented Component

        Description The set of componenets that are implemented in a given system inventory item.

        This element appears inside: inventory-item.

        The implemented-component element has the following contents (in order):

        • component-id attribute (NCName) [required] A reference to a component that is implemented as part of an inventory item.

        • use attribute (string) [optional] The type of implementation

          The value may be locally defined, or one of the following:

          • runs-software: The implemented component is a 'software' component that the inventory item has installed and uses.
          • uses-hardware: The implemented component is a 'hardware' component that the inventory item has installed and uses.
          • enforces-policy: The implemented component is a 'policy' component that the inventory item supports and enforces.
          • implements-process: The implemented component is a 'process' component that the inventory item supports and enforces.

        • prop elements (string) [0 to ∞] A value with a name, attributed to the containing control, part, or group.

        • annotation elements [0 to ∞] A name/value pair with optional explanatory remarks.

        • link elements (markup-line) [0 to ∞] A reference to a local or remote resource

        • responsible-party elements [0 to ∞] A reference to a set of organizations or persons that have responsibility for performing a referenced role relative to the parent context.

          This construct is used to either: 1) associate a party or parties to a role defined on the component using the responsible-role construct, or 2) to define a party or parties that are responsible for a role defined within the context of the containing inventory-item.

        • remarks element (markup-multiline) [0 or 1] Additional commentary on the parent item.

        Description Describes the operational status of the system.

        The status element has the following contents:

        • state attribute (string) [required] The current operating status.

        target

        Name Citation target

        Description An address for retrieval of a citation

        An element (string)

        This element appears inside: citation.

        Remarks

        Semantically equivalent to an href (flag), this is represented as a field in order to permit citations to carry an extra target (for cross-check or fallback) and to permit further qualification (with flags of its own) in future.

        This value is expected to be a URI, and may be an internal link to another object such as a resource documented in back matter.

        When a citation is given as a full (but unparsed) bibliographic citation, use desc instead of title inside this object.
