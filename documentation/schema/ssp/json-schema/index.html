        OSCAL System Security Plan (SSP) JSON Schema Reference

        Schema download oscal_ssp_schema.json

        Schema version 1.0.0-milestone2

        Short name oscal-ssp

        The OSCAL Control SSP format can be used to describe the information typically specified in a system security plan, such as those defined in NIST SP 800-18.

        The root of the OSCAL System Security Plan (SSP) format is system-security-plan.

        annotation

        Name Annotation

        Description A name/value pair with optional explanitory remarks.

        This object appears as a property on: party, person, org, role, responsible-party, system-characteristics, system-information, leveraged-authorization, authorization-boundary, network-architecture, data-flow, system-implementation, user, component, service, interconnection, inventory-item, implemented-component, implemented-requirement, responsible-role, by-component.

        The annotation object has the following members (properties):

        • name property NCName (required)
        • id property string (optional)
        • ns property string (optional)
        • value property string (optional) Indicates the value of the characteristic.

        • remarks object [0 or 1]

          Additional commentary on the parent item.

        Remarks

        Characteristics permit the deployment and management of arbitrary controlled values for any purpose useful to an application or implementation scenario.

        The value of a characteristic is a simple scalar value or list of values.

        The lexical composition of characteristics may be constrained by declarations including matching to regular expressions or by declaring a known datatype.

        Because characteristics are often used as selectors or identifiers for OSCAL operations, their values can be expected frequently to be normalized (e.g., with respect to whitespace) in use; however, this is application defined.

        back-matter

        Name Back matter

        Description A collection of citations and resource references.

        This object appears as a property on: system-security-plan.

        The back-matter object has the following members (properties):

        • Labeled citations, an array containing citation objects, or a single citation object [0 to ∞]

          A citation to resources, either external or internal (by means of internal cross-reference).

        • Labeled resources, an array containing resource objects, or a single resource object [0 to ∞]

          A resource associated with the present document.

        Remarks

        Provides a collection of identified citation and resource objects that can be referenced by a link with a rel value of "reference" and an href value that is a fragment "#" followed by a reference to a citation identifier. A citation can reference a resource by specifying a local target value using the same fragment-identifier approach.

        The following is a contrived example to show the use of link, citation, and resource.

        Example for 'back-matter' not JSONized -- 
            #citation1referenceMy citation
            citation1#resource1resource1https://example.org/some-resource

        citation

        Name Citation

        Description A citation to resources, either external or internal (by means of internal cross-reference).

        This object appears as a property on: back-matter.

        The citation object has the following members (properties):

        • id property string (required)

        • Labeled targets, an array containing target objects, or a single target object [0 to ∞]

          An address for retrieval of a citation

        • title object [0 or 1]

          A title for display and navigation

        • desc object [0 or 1]

          A short textual description

        • Labeled document-ids, an array containing doc-id objects, or a single doc-id object [0 to ∞]

          A document identifier qualified by an identifier type.

        • Labeled properties, an array containing prop objects, or a single prop object [0 to ∞]

          A value with a name, attributed to the containing control, part, or group.

        Remarks

        The structure of a citation is not intended to be a full biblographic representation of a cited resource. Instead, it is expected that matching to a source of bibliograpical information will be handled by use of a doc-id. Use of a canonical identifier such as a DOI is recommended for any citation.

        Within this element, a generic title can cite a title. If citation information takes the form of a bibliographic entry, then desc can be used to provide it.

        component

        Name Component

        Description A defined component that can be part of an implemented system.

        This object appears as a property on: system-implementation.

        The component object has the following members (properties):

        • Implicitly, a property id (type NCName): this value will appear as the key (label) of the parent component JSON object

        • name property string (required) The component's human-readable name.
        • component-type property string (required)

          The value may be locally defined, or one of the following:

          • software: Any software, operating system, or firmware
          • hardware
          • service: A service that may provide APIs.
          • policy
          • process
          • proceedure
          • plan
          • guidance: Any guideline or recommendation
          • standard
          • validation

          A description of the component, including information about its function.

        • Labeled annotations, an array containing annotation objects, or a single annotation object [0 to ∞]

          A name/value pair with optional explanitory remarks.Some specific annotation names can be assigned to the component at large, or can be defined when a component is associated with an inventory-item within the implemented-component construct.

        • Labeled links, an array containing link objects, or a single link object [0 to ∞]

          A reference to a local or remote resource

        • status object [1]

          Describes the operational status of the system.

        • Labeled responsible-role objects [0 to ∞]

          Defines a role that has responsibility for the component.A role defined in this way can be associated with a party or parties here, or where the component is referenced (e.g., inventory-item, implemented-requirement/by-component).The roles "maintainer" or "provider" can be used to define the supplier. In such cases party-id(s) can be used to associate the providing organization. For products, services, and APIs, this is the vendor and/or retailer information. For policies, processes, plans, standards, and guidance, this is the authoring organization.

        • remarks object [0 or 1]

          Additional commentary on the parent item.

        Remarks

        Components may be products, services, application programming interface (APIs), policies, processes, plans, guidance, standards, or other tangible items that enable security and/or privacy.

        The type indicates which of these component types is represented.

        diagram

        Name Diagram

        Description A graphic that provides a visual represenation the system, or some aspect of it.

        This object appears as a property on: authorization-boundary, network-architecture, data-flow.

        The diagram object has the following members (properties):

        • Implicitly, a property id (type NCName): this value will appear as the key (label) of the parent diagram JSON object

          An identifier for this diagram.

        • description object [0 or 1]

          A description of the diagram (e.g., alternate text). This can be used to support compliance with requirements from Section 508 of the United States Workforce Rehabilitation Act of 1973.

        • Labeled properties, an array containing prop objects, or a single prop object [0 to ∞]

          A value with a name, attributed to the containing control, part, or group.

        • Labeled links, an array containing link objects, or a single link object [0 to ∞]

          A reference to a local or remote resource

        • caption object [0 or 1]

          A brief caption to annotate the diagram.

        • remarks object [0 or 1]

          Commentary about the diagram that enhances it.

        Remarks

        A diagram must include a link with a rel value of "diagram", who's href references a remote URI or an internal reference within this document containing the diagram.

        The internal reference "#diagram1" points to an attached resource defined in the back-matter as a resource. The media-type indicates that the image is a Potable Network Graphics (PNG) image.

          { "links" : 
    { "href" : "#diagram1",
      "rel" : "diagram",
      "media-type" : "image\/png",
      "text" : "" } }

        doc-id

        Name Document Identifier

        Description A document identifier qualified by an identifier type.

        An object with property identifier

        This object appears as a property on: metadata, citation.

        The doc-id object must additionally have a (string) property:

        • type property string (required) Qualifies the kind of document identifier.

          The value may be locally defined, or one of the following:

          • doi: A Digital Object Identifier (DOI); use is preferred, since this allows for retrieval of a full bibliographic record.

        Remarks

        A document identifier or name, qualified by a type that indicates the naming system of the identifier. A document identifier may provide a hint to the persistant location of the same or another version of the document.

        Specific usage rules, nomenclatures, and recognized file types (e.g., DOIs, CMS locators, etc.) may be defined at application level. The value of this field may also be a hash or checksum.
